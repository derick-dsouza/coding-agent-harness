- **Shell**: `bash`, `sh`, `zsh`, `exec`, `source`, `export`

- **System**: `sudo`, `su`, `chown`, `chgrp`, `df`, `du`, `mount`

### üí° WORKAROUNDS:
- Instead of `cd path && command`: use `git -C path command` or full paths
- Instead of `npx tsc`: use `node_modules/.bin/tsc` or `npm run <script>`
- Instead of `find`: use `ls -R` or `grep -r` with patterns
- Instead of `sed`/`awk`: use the Read and Edit tools
- Instead of `curl`: use MCP web_search tool or file operations
- Instead of `which command`: check if files exist directly
- Instead of `sort`: use `jq 'sort_by(.field)'` for JSON, or process in Python
- Instead of `tee`: redirect output with `>` or `>>`, or use Create tool for files

### ‚ö° BEST PRACTICES:
1. **Use SDK tools first**: Read, Edit, Create tools are preferred
2. **Use git with -C flag**: `git -C /path/to/repo status` instead of `cd`
3. **Use npm scripts**: Define in package.json, run with `npm run`
4. **Avoid complex pipelines**: Break into multiple commands or use SDK tools
5. **Check existence first**: Use Read tool to verify files exist
